01 / CONCEPT

WHAT IS BITYPE?

BiType is a speculative typeface system constructed entirely on binary logic. Instead of standard bezier contours, every letter, number, and punctuation mark is defined inside a 4x4 coordinate space containing exactly 16 cells.

Each cell holds either a 1 (active vertical stem) or a 0 (active circular ring). The typeface makes digital encoding visible, establishing a visual medium where letterforms double as raw data.

03 / THREE IN ONE

THE TRIPLE NATURE OF THE NAME

  • Bi = 2 (Binary Code): Drawn solely using 1s and 0s.
  • Bi = 2 (2x2 Grid Squares): A 4x4 grid is mathematically equivalent to four 2x2 grid blocks aligned together.
  • Bi = 2 (Dual Typography Layers): The letterform is built directly FROM typography (the 1s and 0s characters/shapes). You are always seeing and using two font layers at the exact same time.
04 / CONTEXT & HYPOTHESIS

DATA EXPOSURE & SURVEILLANCE

Developed as a typography research project (2025–26), BiType operates within the theme of data exposure and surveillance.

"If data collection were made visible through your own image, would you recognise yourself as a tracked entity?"

In our modern ecosystem, data is constantly collected, parsed, and logged behind the scenes. BiType brings this process to light. It poses a visual hypothesis: text that appears to be encrypted patterns or binary static to an untrained eye becomes immediately legible once the user understands the 4x4 reading grid. The system is simultaneously hidden and fully exposed.

05 / THE NO-DOWNLOAD CLAIM

THE GRID IS THE FONT

BiType makes a radical conceptual claim: it is the first font you can use without downloading a file.

Because the font is purely defined by its binary mapping matrix, if you have any two images, shapes, or visual representations arranged in a 4x4 grid, you can construct the entire BiType alphabet. The grid itself is the system. The file is optional; the conceptual framework is universal.

06 / PROCESS & PRODUCTION

TECHNICAL NOTES

The original fonts were generated programmatically using a custom python layout workflow. The parameters utilize UFO structure formats built via defcon and compiled to OpenType/TrueType files with ufo2ft.

Glyph coordinate math begins at a canvas boundary of 1000x1000 units (800 ascender, -200 descender). The 4x4 matrix spans coordinates from left margins of 100u, spacing each cell 200u width/height, applying a internal margins of 20u padding to prevent element overlap.

07 / COLOR PRESERVATION

COLOR FONTS & TECHNICAL LIMITATIONS

While the BiType interface allows live color previewing, standard exported TrueType fonts (.ttf) are restricted to monochrome letterforms. This is a technical boundary of the core OpenType standard.

To preserve custom colors or image tints inside a standalone font file, advanced formats like OpenType-SVG or COLR/CPAL tables are required. Because client-side compilers (such as opentype.js) do not natively compile these color-embedded font tables, downloaded files will always display the binarized vector contours of your custom glyphs in monochrome. To keep full color fidelity, please use the high-resolution vector SVG/PNG export options.
